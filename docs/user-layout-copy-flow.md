# Логика копирования пользовательских укладок

## Главное правило

Пользовательская укладка принадлежит текущему пользователю. Поэтому ее копия может появиться во фронте сразу после нажатия "копировать", даже если синхронизация с сервером произойдет позже.

Отличие от шаблона: пользовательская копия является личной локальной записью и не публикуется для всех. Копия шаблона является опубликованной shared/template записью, поэтому она не показывается как существующая, пока сервер не подтвердил ее создание.

## Копирование пользовательской укладки

1. Пользователь нажимает копирование своей укладки.
2. Фронт создает новую личную укладку в локальном состоянии.
3. Фронт сразу открывает новую укладку для редактирования.
4. Новая укладка попадает в личный список укладок текущего пользователя.
5. Синхронизация отправляет личные изменения на сервер.
6. Если сервер временно недоступен, локальная копия остается у пользователя и будет синхронизирована позже.

## Оффлайн

1. Копирование пользовательской укладки разрешено без связи с сервером.
2. Копия сразу доступна в интерфейсе как личная локальная укладка.
3. До успешной синхронизации сервер еще не является источником этой копии.
4. После восстановления связи фронт отправляет изменения на сервер как обычные личные данные пользователя.

## Запрещено

- Смешивать пользовательскую копию с published shared/template списком.
- Блокировать появление личной копии в интерфейсе до ответа сервера.
- Показывать пользовательскую копию в админском списке шаблонов.
- Переносить оффлайн-правила пользовательских укладок на опубликованные шаблоны.
